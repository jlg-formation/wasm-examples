# Standardisation de WebAssembly par le W3C

> **Question QCM : q2**  
> Quelle organisation est responsable de la standardisation de WebAssembly ?

## Réponse

**Le consortium W3C** (World Wide Web Consortium) est responsable de la standardisation de WebAssembly, en collaboration avec les principaux éditeurs de navigateurs.

## Détails

### Le W3C WebAssembly Working Group

Le [WebAssembly Working Group](https://www.w3.org/wasm/) a été créé en 2017 pour :

- Définir la spécification officielle de WebAssembly
- Assurer l'interopérabilité entre les navigateurs
- Faire évoluer le standard avec de nouvelles fonctionnalités

### Membres fondateurs

Les principaux éditeurs de navigateurs participent activement :

- **Google** (Chrome/V8)
- **Mozilla** (Firefox/SpiderMonkey)
- **Microsoft** (Edge/Chakra puis V8)
- **Apple** (Safari/JavaScriptCore)

### Spécifications officielles

- [WebAssembly Core Specification](https://www.w3.org/TR/wasm-core-1/) - Format binaire et sémantique
- [WebAssembly JavaScript Interface](https://www.w3.org/TR/wasm-js-api-1/) - API JavaScript
- [WebAssembly Web API](https://www.w3.org/TR/wasm-web-api-1/) - Intégration navigateur
