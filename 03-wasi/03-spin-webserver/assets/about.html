Comment l'exemple est organise

1. Spin recoit la requete HTTP

Le trigger HTTP de Spin envoie toutes les routes vers le meme composant WebAssembly.

2. Le composant Rust choisit la reponse

Selon le chemin demande, il renvoie soit une page HTML, soit le CSS, soit le JavaScript, soit l'image, soit la route JSON.

3. Le front appelle l'API locale

Le fichier JavaScript du site consomme `/api/message` avec `fetch()` et injecte la reponse dans la page.

Points pedagogiques

  • Un seul composant Wasm suffit pour un mini site complet.
  • Le manifest `spin.toml` relie le trigger HTTP et le fichier Wasm.
  • Le projet reste lisible pour un public intermediaire.
